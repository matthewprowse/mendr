### F13. [MEDIUM] Potential page import violation — page/components/client.tsx inside page/components/

**Severity:** MEDIUM

- `page.tsx` imports from `./page/components/marketing` (relative import)

**Analysis:**  
This is a structural oddity. The primary route is `page.tsx` at the root (renders the homepage). But there's also a `page/` directory with `components/`. This creates ambiguity:

**Analysis:**  
The `page/components/` directory is route-local to `/page`, which is not a real route. This is confusing. The components should either:
1. Move to `app/components/home-page/` (shared location)
2. Move to a sub-route like `app/landing/components/`
3. Stay where they are but with clear documentation

**Recommendation:**  
Add a README.md to `/app/page/` clarifying that this is **not** the root route (which is `page.tsx`) and explaining why the components directory exists here:
```markdown
# /app/page/ — Components for src/app/page.tsx

These are route-local components for the root homepage. They are organized
in a separate directory because of the large number of sub-components.

This directory does NOT define a route (no page.tsx exists here).
The actual homepage is src/app/page.tsx.
```

## Audit Method

**Tools used:**
- `find` — directory and file enumeration
- `grep -r` — import discovery, pattern matching
- `wc -l` — line counting and complexity analysis
- Manual inspection of large/complex files

**Commands executed:**
```bash
# All routes
find src/app -name "page.tsx" ! -path "*/api/*" | sed 's|.*/src/app/||; s|/page.tsx||'

# Large files
find src/app -type f \( -name "*.tsx" \) ! -path "*/api/*" \
  -exec sh -c 'lines=$(wc -l < "$1"); if [ "$lines" -gt 400 ]; then echo "$lines $1"; fi' _ {} \;

# OS duplicates
find src/app -type f -name "* *" ! -path "*/api/*"

# Empty dirs
find src/app -type d -name "_*" ! -path "*/api/*" -exec sh -c 'count=$(find "$1" -type f | wc -l); [ "$count" -eq 0 ] && echo "$1"' _ {} \;

# Forbidden patterns
grep -r "catch.*: any\|@ts-ignore\|@ts-nocheck\|// TODO\|// FIXME" src/app ! -path "*/api/*"

# framer-motion in hot paths
grep -r "framer-motion" src/app/{diagnosis,processing,report} 2>/dev/null

# Client/server violations
find src/app -name "client.tsx" -exec grep -l "@/lib/auth/supabase-server\|@/lib/auth/admin-auth" {} \;

# Import counts
for file in $(find src/app -name "*.tsx" ! -path "*/api/*"); do
  count=$(grep -c "^import" "$file" 2>/dev/null || true)
  [ "$count" -gt 25 ] && echo "$count $file"
done
```

**Scope covered:**
- ✓ All 39 page.tsx files (100% route coverage)
- ✓ All client components (client.tsx, use-*.ts files)
- ✓ All route-local components and hooks
- ✓ All CSS files
- ✓ File naming conventions (kebab-case, client.tsx, use-*.ts)
- ✓ Forbidden patterns (none found)
- ✓ Import graph (no violations found)
- ✓ Client/server boundary (clean)
- ✓ Large file candidates for extraction

**Not audited (covered by other phases):**
- src/app/api/ (covered by A3)
- src/lib/ (covered by A1)
- src/features/ (covered by A2)
