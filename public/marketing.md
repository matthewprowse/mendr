# Scandio Marketing Spec (Lean MVP)

This is a practical content + design spec for two simple landing pages:

1. Homeowner page (goal: diagnosis starts).
2. Provider page (goal: provider applications).

This version is intentionally **basic**, tied to current product reality, and written to ship fast.

---

## 1) What We Actually Have Today (Do Not Overclaim)

## Homeowner-side real features

- User can start a diagnosis from `Generate Free Scandio Report`.
- User can upload a photo and provide context.
- App generates a diagnosis/report-style output.
- User can share report context (including WhatsApp-ready flow/copy in current page language).
- User can move into matching/contractor discovery flow.
- Geographic positioning is Western Cape.

## Provider-side real features

- Provider-facing join page exists with clear `Apply to Join the Network` CTA.
- Founding network positioning exists.
- No commission messaging exists.
- Provider profile concepts are present (reviews, gallery, credentials, specialisations, contact context).
- Onboarding entry point exists (`/pro/onboard`).

**Rule:** If a claim cannot be demonstrated in the current product, do not put it in hero copy.

---

## 2) Core Positioning (Final Draft Copy)

## Shared value proposition

Scandio helps homeowners diagnose home maintenance issues before they call contractors, then share a structured report so everyone starts from the same problem definition.

## Homeowner promise

Know what is likely wrong, what to ask, and who to contact - before spending money.

## Provider promise

Receive better-informed enquiries from homeowners who already have a diagnosis report in hand.

---

## 3) Page A: Homeowner Marketing Page

## Primary goal

Get users to click `Generate Free Scandio Report`.

## Design direction (simple, not fancy)

- One-column reading flow; avoid complex layouts until conversion data proves need.
- Max width: 720-840px for copy sections.
- Strong vertical rhythm: 64-96px section spacing desktop, 40-56px mobile.
- One primary button style site-wide; same label everywhere.
- Neutral visual system: white/near-white backgrounds, one dark contrast CTA band.
- Use real UI screenshots where possible, placeholders only if unavoidable.

## Section-by-section blueprint (with copy)

### Section A1: Hero

**Purpose**
- Immediate clarity + immediate action.

**Layout**
- H1, short supporting paragraph, primary CTA, one trust line.
- Optional single screenshot to the right on desktop; stacked on mobile.

**Copy (recommended)**
- H1: `Something broken at home? Diagnose it before calling anyone.`
- Body: `Upload a photo, get a clear starting-point diagnosis, and share the same Scandio Report with contractors so you can compare quotes on equal scope.`
- Primary CTA: `Generate Free Scandio Report`
- Secondary text link: `How Scandio works`
- Trust line: `Free report · No account required to start · Western Cape`

**Why this matters**
- 80% of users decide in hero whether to continue or bounce.

---

### Section A2: Problem Framing (short, emotional, plain language)

**Purpose**
- Mirror homeowner pain before explaining product.

**Layout**
- 2-3 short statements, centered, high contrast block.

**Copy (recommended)**
- `Most homeowners do not know what is actually wrong.`
- `That uncertainty leads to random call-outs, unclear quotes, and wasted money.`
- `Scandio gives you a clearer starting point before the first phone call.`

**Why this matters**
- Improves relevance and reduces skepticism.

---

### Section A3: How It Works (3 steps only)

**Purpose**
- Reduce perceived complexity.

**Layout**
- 3 stacked steps with small screenshot or icon each.

**Copy (recommended)**
1. `Capture the issue`  
   `Take a photo and add context in plain language.`
2. `Get your Scandio Report`  
   `Receive a structured diagnosis summary in under a minute.`
3. `Share and compare`  
   `Send the report to contractors and compare quotes on the same scope.`

**Why this matters**
- Converts uncertain users who need process clarity.

---

### Section A4: Key Benefits (4 cards max)

**Purpose**
- Show outcome value, not technical detail.

**Layout**
- 2x2 card grid desktop, stacked mobile.

**Copy (recommended card titles + body)**
1. `Less guesswork` - `Understand likely fault patterns before booking visits.`
2. `Fairer quote comparison` - `Give every contractor the same report context.`
3. `Faster first conversations` - `Contractors can respond with better context.`
4. `You control sharing` - `Share your report only with people you choose.`

**Why this matters**
- Scannable proof of practical value.

---

### Section A5: Product Proof (single screenshot strip)

**Purpose**
- Make product feel real and trustworthy.

**Layout**
- 1-3 screenshots with one-line caption each.

**Copy (caption examples)**
- `Diagnosis summary with likely issue and next-step context.`
- `Shareable report format for contractor conversations.`
- `Match flow for finding nearby providers.`

**Why this matters**
- Visual evidence boosts conversion for cold traffic.

---

### Section A6: FAQ (minimum 5 questions)

**Purpose**
- Handle objections + capture long-tail SEO.

**Required questions**
1. Is the Scandio Report free?
2. How accurate is the diagnosis?
3. Can I share with multiple contractors?
4. Which areas do you cover?
5. Is my report private?

**Why this matters**
- FAQ is both conversion support and SEO asset.

---

### Section A7: Final CTA Band

**Purpose**
- Catch late-stage scroll users.

**Layout**
- Dark background, one sentence, one button.

**Copy (recommended)**
- Heading: `Find out what is likely wrong before you pay for call-outs.`
- Subtext: `Free. Fast. Built for Western Cape homeowners.`
- CTA: `Generate Free Scandio Report`

**Why this matters**
- End-of-page conversion recapture.

## 5) Visual + UX Rules (for both pages)

## Keep it clean

- Use 1 primary font family and 1 heading scale.
- Keep paragraphs to 1-3 lines.
- Limit line length for readability.
- Use plenty of whitespace instead of decorative complexity.

## Keep it consistent

- Same CTA wording in hero and final CTA.
- Same button hierarchy across sections.
- Same trust line pattern on both pages.

## Keep it honest

- Avoid words like "best", "guaranteed", "vetted" unless strictly verifiable.
- Prefer "likely", "starting point", "informed" wording.

# Scandio Marketing Blueprint

This document defines what each marketing page should communicate, how sections should be structured, and why each section matters for conversion and SEO.

## Product Positioning

### Core value proposition

Scandio helps homeowners diagnose home maintenance issues quickly and connect with relevant local contractors using a shared, structured report.

### The two-sided marketplace promise

- Homeowners get clarity before spending money.
- Providers get better-qualified leads with less wasted quoting time.

### Primary conversion goals (must drive page structure)

1. Homeowner page: drive users to `Generate Free Scandio Report` (diagnosis flow).
2. Provider page: drive users to `Apply to Join the Network` / join the waitlist or founding network.

## Audience Benefits

## Benefits for homeowners

- Faster understanding of faults (less guesswork and panic).
- Better quote quality through shared problem context.
- Easier comparison between contractors quoting on the same scope.
- Safer decision-making with transparent provider profiles and reviews.
- Control and ownership of the report (share link/WhatsApp summary).

## Benefits for companies/providers

- Pre-diagnosed, intent-rich enquiries instead of generic cold leads.
- Higher first-visit fix potential due to better pre-visit context.
- Profile visibility in local search inside the platform.
- Trust-building assets (reviews, gallery, credentials, specialisations).
- Business model clarity (0% commission positioning, clear pricing roadmap).

## Flows and Feature Map

## Homeowner flow (north star: diagnosis completion)

1. Land on homeowner marketing page.
2. Understand value in first screen.
3. Click `Generate Free Scandio Report`.
4. Upload image/context and run diagnosis.
5. View report with suggested trade context.
6. Share report and/or continue to provider matching.

### Homeowner features to highlight

- AI diagnosis from photo + context.
- Structured report output.
- Shareable report link and WhatsApp-ready summary.
- Provider matching by trade/location/availability.
- Privacy control over who can view report.

## Provider flow (north star: network application)

1. Land on provider marketing page.
2. Understand lead quality and commercial upside.
3. Click `Apply to Join the Network`.
4. Submit onboarding/application details.
5. Set up provider profile.
6. Start receiving informed enquiries tied to reports.

### Provider features to highlight

- Informed leads from homeowners with pre-work diagnosis context.
- Profile page with services, proof, and trust signals.
- Reviews and gallery for credibility.
- Local discovery within service area.
- Clear pricing pathway (founding/free period, later paid tiers).

## Page 1: Homeowner Marketing Page

Use this structure in order. Each section includes **what to include** and **why it matters**.

### 1) Hero (above the fold)

**What to include**
- Headline about diagnosing home issues fast and confidently.
- Subheading clarifying this is a starting point, not a physical inspection replacement.
- Primary CTA button: `Generate Free Scandio Report`.
- Secondary CTA: optional `See how it works`.
- Trust micro-copy (region served, free report, privacy signal).

**Why this is important**
- Sets relevance in under 5 seconds.
- Anchors the main conversion action immediately.
- Reduces early skepticism with clear expectation-setting.

### 2) Problem -> solution framing

**What to include**
- Common homeowner pain points: uncertainty, inconsistent quotes, wasted call-outs.
- How Scandio changes the process: diagnose first, then compare providers on shared scope.

**Why this is important**
- Great UX copy mirrors user frustration before presenting features.
- Improves motivation to continue scrolling toward CTA.

### 3) How it works (3 simple steps)

**What to include**
- Step 1: upload photo and describe issue.
- Step 2: receive structured diagnosis/report.
- Step 3: share report and find relevant contractors.

**Why this is important**
- Lowers perceived effort.
- Turns abstract AI promise into an understandable process.

### 4) Core benefits grid

**What to include**
- 4-6 benefit cards focused on outcomes (not technical implementation).
- Suggested themes: clarity, faster fixes, fair quote comparison, confidence, data ownership.

**Why this is important**
- Scannable layout for fast evaluators.
- Reinforces value at different buyer motivations.

### 5) Product proof / screenshots / report preview

**What to include**
- UI snapshots of diagnosis output, share function, and provider match experience.
- Short captions tied to user outcomes.

**Why this is important**
- Visual proof decreases "too good to be true" friction.
- Supports high-UX expectations from design-conscious users.

### 6) Social proof and trust signals

**What to include**
- Geographic scope (Western Cape), usage metrics when available, testimonials/review snippets.
- Security/privacy statement (report access controlled by homeowner sharing).

**Why this is important**
- Trust is essential for home and money-related decisions.
- Strengthens conversion when traffic comes from SEO (colder intent).

### 7) FAQ (SEO + objection handling)

**What to include**
- Accuracy and limitations.
- Cost/pricing.
- How sharing works.
- Coverage area.
- Data privacy.

**Why this is important**
- Removes decision blockers near conversion.
- FAQ content captures long-tail search queries.

### 8) Final conversion band

**What to include**
- Strong restatement of value.
- Repeated primary CTA: `Generate Free Scandio Report`.

**Why this is important**
- Many users convert after full-page review.
- Gives a clean end-point action without forcing upward scroll.

## Page 2: Provider / Company Marketing Page

Use this structure in order. Each section includes **what to include** and **why it matters**.

### 1) Hero (commercial value first)

**What to include**

- Subheading explaining pre-diagnosed homeowner leads.
- Primary CTA: `Apply to Join the Network`.
- Trust micro-copy: free to join (if current), 0% commission, service region.

**Why this is important**

- Directly maps to provider intent and increases signup likelihood.

**What to include**
- Founding network benefits (early visibility, early profile advantage, lock-in incentives if used).
- Clear statement of roadmap/pricing timeline.

**Why this is important**
- Creates urgency without manipulative copy.
- Helps serious providers self-qualify quickly.

### 3) How it works for providers (3 steps)

**What to include**
- Apply.
- Complete profile.
- Receive informed enquiries with Scandio reports.

**Why this is important**
- Reduces uncertainty around onboarding effort.
- Makes signup action feel manageable.

### 4) Provider benefit pillars

**What to include**
- Qualified leads.
- Better first-visit prep.
- Trust and credibility through profile depth.
- Local discoverability.
- Transparent commercial model.

**Why this is important**
- Turns platform features into clear business outcomes.
- Helps independent contractors and teams evaluate fit quickly.

### 5) Pricing/transparency section

**What to include**
- Current plan status (e.g., founding phase free).
- Future plan structure (indicative tiers).
- No hidden fees / no commissions messaging.

**Why this is important**
- Price ambiguity can kill signup conversion.
- Clear economics builds brand trust early.

### 6) Credibility + expected outcomes

**What to include**
- "What to expect" bullets (lead quality, no commission, visibility model).
- Future case studies/testimonials once available.

**Why this is important**
- Handles realistic expectations.

### 7) Final CTA (single action)

**What to include**
- Restate provider upside in one line.
- Repeat CTA: `Apply to Join the Network`.

**Why this is important**
- Minimizes decision friction at the end of the page.
- Keeps page focused on one measurable conversion event.

## SEO Priorities (Bootstrap-Friendly)

Focus on high-leverage, low-cost SEO execution.

### On-page priorities

- One clear H1 per page with intent keywords.
- Intent-aligned H2 sections (diagnosis, quote comparison, local contractors, contractor leads).
- Internal links between homeowner page, provider page, FAQ, contact, and core flows.
- Strong metadata (title + description) aligned with conversion and region.
- Image alt text that describes UI and user outcome.

### Content priorities

- FAQ sections for long-tail searches.
- Local modifiers (Western Cape, Cape Town, Stellenbosch, Paarl, Somerset West where accurate).
- Plain-language copy answering "what", "how", "cost", "coverage", "privacy".
- Keep technical claims precise; avoid unverifiable superlatives.

### Structured data priorities

- Homeowner page: `FAQPage`.
- Provider page: `FAQPage` and `Organization` basics where appropriate.

### Measurement priorities

- Track hero CTA clicks, scroll depth, and final CTA clicks.
- Track diagnosis starts/completions from homeowner page.
- Track provider application starts/completions from provider page.
- Use these events to iterate copy before adding paid spend.

### Relevance score (for your use case)

- Conversion relevance: **Medium** (6/10).
- Trust relevance: **Medium-High** (7/10) when claims are real and specific.
- SEO relevance: **Low** (2/10), because badges are mostly visual and provide limited indexable value.

Use badges sparingly: one to three max near hero/supporting proof. They should not compete with the main CTA.

## Copy and UX Principles for This Product

- One page, one primary action.
- Write outcome-first copy ("what user gets") before feature copy ("how it works").
- Repeat CTA consistently with the same wording.
- Keep form friction low; ask only essential fields first.
- Pair every claim with proof (UI example, policy, metric, or testimonial).
- Design for skimming: short paragraphs, clear hierarchy, strong section labels.

## MVP Section Checklist

If implementation time is limited, prioritize these sections first:

1. Hero with clear CTA.
2. 3-step how-it-works.
3. Benefits grid (outcome focused).
4. FAQ (top objections).
5. Final CTA band.

Everything else can be iterated after conversion data starts coming in.
